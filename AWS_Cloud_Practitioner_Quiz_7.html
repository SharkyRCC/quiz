Q1 Which AWS service provides the ability to create custom security rules for your specific use case?

üîê Explanation:

All of these services allow custom security rules: GuardDuty supports custom threat detection rules, AWS Config allows custom compliance rules, and AWS Security Hub supports custom security standards and findings. Each provides flexibility for specific security requirements.

Q2 A company uses Amazon S3 to store records that can contain personally identifiable information (PII). The company wants a solution that can monitor all S3 buckets for PII and immediately alert staff about vulnerabilities. Which AWS service will meet these requirements?

üîê Explanation:

Amazon Macie is specifically designed to discover and protect sensitive data, including PII, stored in Amazon S3. Macie continuously monitors S3 buckets, identifies PII using machine learning and pattern matching, and generates alerts when vulnerabilities or data security risks are detected.

Q3 Which AWS service provides real-time threat intelligence feeds?

üîê Explanation:

Amazon GuardDuty uses real-time threat intelligence feeds from AWS, CrowdStrike, and Proofpoint to enhance its threat detection capabilities. These feeds help GuardDuty identify known malicious IP addresses, domains, and other threat indicators.

Q4 A company wants to ensure their AWS resources comply with CIS benchmarks. Which service can help with this?

üîê Explanation:

AWS Security Hub includes security standards like CIS AWS Foundations Benchmark. It automatically runs security checks against these standards and provides a compliance score, helping you ensure your AWS resources comply with industry benchmarks and best practices.

Q5 Which AWS service can help you implement continuous compliance monitoring?

üîê Explanation:

All of these services support continuous compliance monitoring: AWS Config continuously evaluates resource configurations against compliance rules, AWS Security Hub provides ongoing compliance scoring against security standards, and AWS Systems Manager Compliance tracks patch compliance and other system-level compliance requirements.

Q6 What is the main difference between AWS Shield Standard and AWS Shield Advanced?

üîê Explanation:

AWS Shield Standard is free and automatically enabled for all AWS customers, providing basic DDoS protection. AWS Shield Advanced is a paid service that provides enhanced DDoS protection, 24/7 access to the DDoS Response Team, and advanced attack diagnostics.

Q7 Which AWS service or feature can a user configure to limit network access at the subnet level?

üîê Explanation:

Network Access Control Lists (NACLs) are the primary mechanism for controlling inbound and outbound network traffic at the subnet level within a VPC. They act as a stateless firewall, evaluating traffic entering and exiting a subnet against defined rules.

Q8 A company website is experiencing DDoS attacks. Which AWS service can help protect the company website against these attacks?

üîê Explanation:

AWS Shield is designed specifically to protect applications running on AWS from DDoS attacks. It provides always-on detection and automatic inline mitigations that minimize application downtime and latency. AWS Shield comes in two tiers: Standard and Advanced.

Q9 Which AWS service can help you create a security baseline for your AWS accounts?

üîê Explanation:

AWS Control Tower helps you set up and govern a secure, multi-account AWS environment based on best practices. It provides pre-configured security baselines (guardrails) and automates the setup of a landing zone with security and compliance controls.

Q10 Which AWS service can help identify IAM users who have not enabled multi-factor authentication (MFA)?

üîê Explanation:

IAM credential reports provide a comprehensive overview of the status of various security credentials for all IAM users in an AWS account. This includes details about password usage, access keys, and the status of MFA, allowing administrators to identify users who do not have MFA enabled.

Q11 What is the main advantage of using AWS Secrets Manager over storing secrets in environment variables?

üîê Explanation:

The main advantage of AWS Secrets Manager is automatic rotation and encryption of secrets. Unlike environment variables, Secrets Manager provides secure storage, automatic rotation capabilities, fine-grained access control, and audit trails for secret access.

Q12 What is the primary purpose of Service Control Policies (SCPs) in AWS Organizations?

üîê Explanation:

Service Control Policies (SCPs) in AWS Organizations serve as security guardrails and permission boundaries. They define the maximum permissions that can be granted to users and roles in member accounts, helping enforce security policies across the entire organization.

Q13 Which AWS service can help you understand the blast radius of a security incident?

üîê Explanation:

Amazon Detective analyzes log data to investigate security findings and conduct root cause analysis. It helps you understand the blast radius of a security incident by visualizing the relationships between AWS resources and activities, making it easier to determine what was affected.

Q14 Which AWS service can help you manage and rotate database credentials automatically?

üîê Explanation:

AWS Secrets Manager helps you manage, retrieve, and rotate database credentials, API keys, and other secrets throughout their lifecycle. It can automatically rotate credentials for supported databases like RDS, reducing the operational burden of manual credential management.

Q15 A company wants to use a managed service to identify and protect sensitive data that is stored in Amazon S3. Which AWS service will meet these requirements?

üîê Explanation:

Amazon Macie uses machine learning and pattern matching to discover, classify, and protect sensitive data, such as personally identifiable information (PII) or protected health information (PHI), stored in S3. It automates the process of data discovery and provides alerts and dashboards to monitor data security posture.

Q16 Which AWS service can automatically block IP addresses that are generating malicious traffic?

üîê Explanation:

AWS WAF can automatically block IP addresses based on various conditions including rate limiting, geographic restrictions, and integration with threat intelligence feeds. It can dynamically update rules to block malicious IP addresses generating suspicious traffic.

Q17 A company needs to encrypt data at rest in Amazon S3. Who is responsible for managing this encryption according to the AWS shared responsibility model?

üîê Explanation:

According to the AWS shared responsibility model, the customer is responsible for data encryption at rest. This includes choosing the encryption method, managing encryption keys, and configuring encryption settings. AWS provides the tools and services, but the customer controls the implementation.

Q18 Which AWS service can help you identify and respond to potential insider threats?

üîê Explanation:

All of these services can help with insider threat detection: GuardDuty detects unusual behavior patterns, CloudTrail provides detailed audit logs of user activities, and Detective helps investigate and analyze suspicious activities. They work together to provide comprehensive insider threat detection.

Q19 A company needs to ensure that all S3 buckets are encrypted. Which AWS service can help monitor and enforce this requirement?

üîê Explanation:

AWS Config can monitor S3 bucket encryption settings using managed rules like 's3-bucket-server-side-encryption-enabled'. It can detect non-compliant buckets and trigger automatic remediation actions to ensure all S3 buckets are encrypted.

Q20 Which AWS service provides centralized management of firewall rules across multiple AWS accounts?

üîê Explanation:

AWS Firewall Manager is used to centrally manage firewall rules across multiple AWS accounts and resources, providing a single pane of glass for security policy enforcement. It helps you configure and manage firewall rules across your accounts and applications.

Q21 A company wants to monitor for data exfiltration attempts. Which AWS service would be most appropriate?

üîê Explanation:

Amazon GuardDuty can detect data exfiltration attempts by analyzing network traffic patterns, DNS queries, and other behavioral indicators. It uses machine learning to identify unusual data transfer patterns that might indicate unauthorized data exfiltration.

Q22 A company wants to automatically remediate security findings. Which AWS service can help with this?

üîê Explanation:

AWS Config Rules can automatically remediate non-compliant resources when combined with AWS Systems Manager Automation documents or AWS Lambda functions. Config Rules evaluate whether your AWS resources comply with your desired configurations and can trigger automatic remediation actions.

Q23 A company wants to ensure that all their AWS resources are tagged according to their security policy. Which service can help enforce this?

üîê Explanation:

AWS Config can enforce tagging policies using Config Rules. It can automatically detect resources that don't comply with tagging requirements and trigger remediation actions to ensure all resources are properly tagged according to security policies.

Q24 A company needs to ensure that sensitive data is automatically discovered and classified across their AWS environment. Which service combination would be most effective?

üîê Explanation:

Amazon Macie automatically discovers and classifies sensitive data in S3, while AWS Config can monitor and ensure compliance with data classification policies. This combination provides comprehensive data discovery, classification, and governance capabilities.

üîê Explanation:

AWS Artifact provides access to AWS compliance reports and agreements, including HIPAA compliance documentation. It offers guidance on how to configure AWS services to meet HIPAA requirements and provides the necessary compliance artifacts for audits.

Q26 Which AWS service can be used to retrieve compliance reports on demand?

üîê Explanation:

Q27 Which AWS service can help you implement data residency requirements for compliance?

üîê Explanation:

AWS Regions help implement data residency requirements by allowing you to choose specific geographic locations where your data is stored and processed. This helps meet compliance requirements that mandate data must remain within certain geographic boundaries.

Q28 What is the primary purpose of AWS CloudTrail?

üîê Explanation:

AWS CloudTrail's primary purpose is to enable traceability by providing a detailed audit trail of who did what, when, and from where within your AWS infrastructure. It records API calls and user activities, which is essential for security investigations, compliance reporting, and troubleshooting.

Q29 Which AWS service provides automated security assessments for container images?

üîê Explanation:

Amazon Inspector provides automated security assessments for both EC2 instances and container images. It can scan container images stored in Amazon ECR for software vulnerabilities and provide detailed findings to help improve security posture.

Q30 What is the primary security benefit of using AWS Systems Manager Parameter Store?

üîê Explanation:

The primary security benefit of AWS Systems Manager Parameter Store is secure storage of configuration data and secrets. It provides encryption at rest, fine-grained access control, and audit trails for parameter access, making it more secure than storing sensitive data in code or configuration files.

Q31 Which AWS service can help you implement network segmentation in your VPC?

üîê Explanation:

Both Security Groups and Network ACLs can help implement network segmentation in your VPC. Security Groups provide instance-level segmentation, while Network ACLs provide subnet-level segmentation. Using both together provides defense in depth.

Q32 What is the difference between Security Groups and Network ACLs?

üîê Explanation:

Security Groups are stateful, meaning they automatically allow return traffic for established connections. Network ACLs are stateless, meaning they evaluate each packet independently and require explicit rules for both inbound and outbound traffic.

Q33 What is the purpose of AWS Systems Manager Session Manager?

üîê Explanation:

AWS Systems Manager Session Manager provides secure shell access to EC2 instances without the need to open inbound ports, manage SSH keys, or use bastion hosts. It provides a secure way to access instances while maintaining full audit trails.

Q34 What type of security control are Security Groups in AWS?

üîê Explanation:

Security Groups are preventive security controls because they prevent unauthorized network traffic from reaching EC2 instances by filtering traffic based on predefined rules. They act as virtual firewalls that block unwanted traffic before it can access your instances.

Q35 Which AWS service provides the most comprehensive view of your security posture across all AWS accounts?

üîê Explanation:

AWS Security Hub provides the most comprehensive view of your security posture by aggregating security findings from multiple AWS security services and third-party tools across all your AWS accounts. It provides a centralized dashboard for security management.

Q36 Which AWS service can help you implement zero-trust network access for your applications?

üîê Explanation:

AWS PrivateLink enables private connectivity between VPCs and AWS services without exposing traffic to the public internet. It supports zero-trust network access by providing secure, private connections that don't rely on network location for security decisions.

Q37 What is the purpose of AWS Resource Access Manager (RAM)?

üîê Explanation:

AWS Resource Access Manager (RAM) enables you to securely share AWS resources across AWS accounts within your organization. It helps you avoid resource duplication and provides centralized management of shared resources while maintaining security boundaries.

Q38 Which AWS service can help you identify unused IAM roles and users?

üîê Explanation:

AWS IAM Access Analyzer can help identify unused IAM roles and users by analyzing access patterns and last used information. It provides insights into which IAM entities are not being used, helping you clean up unnecessary permissions and reduce security risks.

Q39 Which AWS service can help you implement data loss prevention (DLP) for sensitive data in S3?

üîê Explanation:

Amazon Macie can help implement data loss prevention by continuously monitoring S3 buckets for sensitive data like PII, PHI, and financial information. It can detect and alert on policy violations and unauthorized access to sensitive data.

Q40 A company runs many Amazon EC2 instances in its VPC. The company wants to use a native AWS security resource to control network traffic between certain EC2 instances. Which AWS service or feature will meet this requirement?

üîê Explanation:

Security Groups act as virtual firewalls for your EC2 instances, controlling inbound and outbound traffic at the instance level. They use a stateful model and are associated with EC2 instances, allowing you to control traffic based on protocols, port numbers, and source/destination IP addresses.

Q41 What is the primary purpose of AWS CloudHSM?

üîê Explanation:

AWS CloudHSM provides hardware security modules (HSMs) in the cloud for generating and using encryption keys. It offers dedicated hardware for cryptographic operations and key management, providing higher security assurance than software-based solutions.

Q42 Which AWS service can help you automate incident response procedures?

üîê Explanation:
